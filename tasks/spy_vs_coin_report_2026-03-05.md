# SPY vs Coin API 乖離調査レポート (2026-03-05)

## 結論

SPYデータ（spy_messages）はチャット内tipのみを検出する設計であり、Coin API（coin_transactions）の全売上の**8.4%しか捕捉できない**。これは設計上の仕様であり、バグではない。

---

## 1. データソースの違い

| 観点 | spy_messages | coin_transactions |
|---|---|---|
| データ元 | Centrifugo WebSocket（リアルタイム） | Stripchat Earnings API（定期同期） |
| 検出対象 | チャット内 tip のみ | tip/ticketShow/studio/private/group/spy/photo/gift 等全タイプ |
| 同期タイミング | リアルタイム | Chrome拡張から手動実行 |
| 用途 | 配信中のリアルタイム監視 | 正確な売上集計・P/L計算 |

## 4. 発見した問題点

### Critical: SPY監視ギャップ（8日/17日が欠損）
- 2/16, 2/18, 2/19, 2/23, 2/24, 2/25, 2/26, 3/2 の**8日間**がSPYデータ0件
- Collector停止やWebSocket切断が原因
- この間もcoin_transactionsではデータが記録されている

### Critical: hanshakun SPY > Coin tip の逆転
- SPY tip: 21,232 tk vs Coin tip: 6,690 tk → SPYが3.17倍
- 原因推定: SPYがticketShowのチップもtipとしてカウントしている可能性
  - hanshakun Coin ticketShow: 208,248 tk（全売上の96.8%）
  - WebSocketメッセージでticketShow中のチップがmsg_type=tipとして記録されている可能性

### High: sessions重複セッション 30件
- 同一キャストで5分以内に開始した重複セッションが30件
- 主に Kyokon_King(14件), _noanyan_(8件), rika_secret(3件), fantasy_JP(3件), 0o_MOMO_o0(2件)
- Collectorのステータスポーリング間隔（5秒）による誤検出

### Medium: studioタイプの大口取引
- Risa_06の売上の77.5%が`studio`タイプ（336,337 tk）
- SPYでは一切検出できない収入源
- 2/16: 48,329tk, 2/23: 163,165tk, 3/2: 124,873tk（不定期だが大口）

### Info: SPY最終活動日
- hanshakun: 2026-02-22 が最後（2週間以上SPYデータなし）
- Risa_06: 2026-03-04 が最後（前日活動あり）

## 5. 乖離の内訳（設計上の仕様）

coin_transactionsの649,277 tkの内訳:
- **studio**: 大部分（推定40%以上）→ SPYでは検出不可
- **ticketShow**: 約40%（推定）→ SPYでは検出不可
- **tip**: 約8%（50,459 tk）→ SPYで検出可能（ただし監視中のみ）
- **その他**(private/group/spy/photo等): 約2%

SPYが捕捉できるのはtipタイプのみ、かつCollector稼働中のみ。

## 6. 推奨アクション

### 即時対応
1. **売上表示の明確化**: UIで「SPY売上（リアルタイム推定）」と「Coin API売上（確定値）」を明確に区別
2. **SPY売上をP/L計算に使わない**: セッションP/LはCoin API値のみで計算すべき（現在のget_session_plは正しい）

### 中期対応
3. **SPY監視の安定化**: Collector常駐化（pm2）+ 切断検出→自動再接続
4. **hanshakun tip逆転の原因調査**: ticketShow中のチップがSPYでどう記録されるか、Collectorのchat.tsパーサーを検証
5. **sessions重複の自動マージ**: 5分以内の重複セッションを自動統合するバッチ処理

### 長期対応
6. **studioタイプの可視化**: studio(プライベート配信)の売上をダッシュボードで目立つ位置に表示
7. **SPY監視稼働率の可視化**: `/admin/health`にSPY稼働率（日別データ有無率）を追加
