## 1. DM送信機能

SLSのDM送信は、課金実績に基づいてユーザーをセグメント分けし、セグメントごとにメッセージを一括送信する機能です。
送信はStripchat APIを経由して行われ、すべての送信にキャンペーンIDが付与されるため、後から効果測定できます。

### 1.1 セグメント別DM送信画面を開く

1. サイドバーから「キャスト」をクリック
2. DM送信したいキャスト（例: `Risa_06`）をクリック
3. 上部タブから「DM」タブをクリック
   - URL: `/casts/Risa_06?tab=dm`

画面が開くと、セグメント選択パネルが表示されます。

---

### 1.2 セグメントの意味

SLSでは、ユーザーの課金額に基づいて5つのセグメントに分類しています。
データソースは `coin_transactions`（Coin API）で、SPYチャットログではなく実際の課金履歴が基準です。

| セグメント | アイコン | コイン範囲 | どんなユーザーか |
|---|---|---|---|
| **Whale** | 🐋 | 3,000tk 以上 | 最高額の支援者。最重要顧客 |
| **VIP** | ⭐ | 500〜2,999tk | 継続的に応援してくれるVIP層 |
| **Regular** | 👤 | 150〜499tk | 定期的に少額課金する常連 |
| **Light** | 🌱 | 1〜149tk | 一度だけ少額課金した初期ユーザー |
| **Churned** | 💤 | 過去に課金あり、選択期間内はゼロ | 離脱ユーザー（最近来なくなった人） |

#### Churned（離脱）の特殊な動き

「Churned」を選択すると、自動で期間が「過去30日」に切り替わります。
これは「全期間で課金履歴があるが、直近30日間は0円の人」を検出するためです。

> **例**: 過去に500tk課金したが、直近30日間は0円 → Churned として抽出

---

### 1.3 期間フィルタ

コイン集計の対象期間を5つから選べます。

| ボタン | 対象期間 | 使いどころ |
|---|---|---|
| **過去7日** | 直近7日間 | 今週の配信に来てくれた人に送りたいとき |
| **過去30日** | 直近30日間 | 今月のアクティブユーザーに送りたいとき |
| **過去90日** | 直近90日間 | 四半期単位で見たいとき |
| **全期間** | 2025-02-15〜今日 | 累計課金額で判断したいとき |
| **カスタム** | 開始日〜終了日を手動指定 | 特定イベント期間で絞りたいとき |

#### カスタム期間の使い方

1. 「カスタム」ボタンをクリック
2. 「開始日」の日付入力欄に開始日を入力（例: `2026-02-01`）
3. 「終了日」の日付入力欄に終了日を入力（例: `2026-02-28`）
4. Enterキーを押すと集計が更新される

---

### 1.4 コインレンジフィルタ

セグメントのプリセット以外にも、コインの最小値・最大値を手動で指定できます。

- **最小コイン**: デフォルト `1`（0tkユーザーを除外）
- **最大コイン**: デフォルト `999,999`

#### 使用例

| やりたいこと | 設定 |
|---|---|
| 1,000tk以上の太客だけに送りたい | 最小: `1000` / 最大: `999999` |
| 100〜500tkの中間層だけ | 最小: `100` / 最大: `500` |
| チケットショー客っぽい人（150〜300tk、少回数） | 最小: `150` / 最大: `300` |

> **ヒント**: プリセットボタン（Whale, VIP等）をクリックすると、コインレンジが自動設定されます。
> プリセットを選択した後に手動でレンジを変更すると、プリセットが解除されて「カスタム」状態になります。

---

### 1.5 該当ユーザーリストの確認

フィルタを設定すると、画面に以下が表示されます。

| 表示項目 | 説明 |
|---|---|
| **該当ユーザー数** | フィルタに合致する人数（リアルタイム更新） |
| **合計コイン** | 該当ユーザーの選択期間内コイン合計 |
| **ユーザーリスト** | 上位30名をコイン額の降順で表示 |

各ユーザーの横に以下が表示されます:
- ユーザー名
- 累計コイン数
- 🚫ボタン（ブラックリストに追加）

> **重要**: ユーザーリストをスクロールして、意図しないユーザーが含まれていないか必ず確認してください。

---

### 1.6 ブラックリスト管理

特定のユーザーにDMを送りたくない場合、ブラックリストに登録できます。
ブラックリストに登録されたユーザーは、すべてのセグメントから自動で除外されます。

#### ブラックリストの開き方

ユーザーリストの下にある「ブラックリスト（N名）」アコーディオンをクリックして展開します。

#### 個別追加

1. ユーザーリスト内の、除外したいユーザーの右にある 🚫 ボタンをクリック
2. 即座にブラックリストに追加され、送信対象から除外される

#### 一括追加

1. ブラックリストパネル内のテキスト入力欄に、ユーザー名をカンマ区切りで入力
   - 例: `user123, baduser456, spammer789`
2. 「追加」ボタンをクリック

#### ブラックリスト解除

1. ブラックリストパネル内の該当ユーザー横の「解除」ボタンをクリック

> **注意**: ブラックリストはキャストごとに管理されます。
> Risa_06のブラックリストにuser123を入れても、hanshakunのDMには影響しません。

### 1.7 メッセージ設定から送信まで

DM送信は5つのステップで行います。

#### ステップ1: セグメント + フィルタを設定する

1. 期間フィルタを選択（例: 過去30日）
2. プリセットをクリック（例: VIP）、またはコインレンジを手動入力
3. ユーザーリストを確認

#### ステップ2: メッセージを入力する

テキスト入力欄にDMのメッセージを入力します。

- **`{username}` プレースホルダ**: メッセージ内に `{username}` と書くと、送信時に各ユーザーの名前に自動置換されます

プリセット（Whale, VIP等）を選択した場合、デフォルトのテンプレートが自動入力されます。

| セグメント | デフォルトテンプレート |
|---|---|
| Whale | `{username}さん、いつも本当にありがとうございます！{username}さんのおかげで毎日頑張れています。また遊びに来てくれたら嬉しいな！` |
| VIP | `{username}さん、応援ありがとうございます！{username}さんが来てくれるだけで嬉しいです。またお話しましょう！` |
| Regular | `{username}さん、こんにちは！最近配信に来てくれてありがとう。また気が向いたら遊びに来てね！` |
| Light | `{username}さん、はじめまして（かな？）！来てくれてありがとう。また会えたら嬉しいな！` |
| Churned | `{username}さん、最近見かけなくて寂しいです...！また遊びに来てくれたら嬉しいな。待ってるね！` |

テンプレートはそのまま使っても、自由に編集してもOKです。

#### ステップ3: キャンペーン名を入力する（任意だが推奨）

キャンペーン名を入力しておくと、後からDM効果を測定できます。

- 例: `3月復帰DM`, `新規お礼`, `チケチャフォロー`
- 未入力の場合は `segment_dm` が自動設定される
- 送信時に自動で `seg_whale_3月復帰DM_20260306_1456` のような形式に整形される

#### ステップ4: プレビュー確認する

1. 「プレビュー確認 →」ボタンをクリック
2. モーダルが開き、以下を確認できる:
   - **送信対象数**: 何名に送るか
   - **セグメント名**: どのセグメントか
   - **期間フィルタ**: いつからいつまでの課金データか
   - **コインレンジ**: 何tk〜何tkで絞っているか
   - **メッセージ例**: 1名目のユーザー名で `{username}` を置換した実際の文面
   - **対象ユーザー一覧**: 最大100名のユーザー名タグ（それ以上は「...他XXX名」）

> **必ず確認すること**: 送信対象数が想定通りか、メッセージの文面に誤りがないか

#### ステップ5: 送信を実行する

1. プレビューモーダル内の「NNN名にDM送信を実行」ボタンをクリック
2. 処理が開始される:
   - ユーザーリストを500件ずつ `dm_send_log` テーブルにキュー登録
   - ステータスは `queued`（待機中）で登録される
3. 結果メッセージが表示される:
   - 成功: 「NNN件のDMをキューに登録しました」
   - 一部失敗: 成功件数 + エラー件数が表示される

> **注意**: 「送信を実行」ボタンを押した時点では、まだDMは送られていません。
> キューに登録されただけで、実際の送信はバックグラウンドのDMサービスが3秒間隔で1件ずつ処理します。

### 1.8 DM送信の安全機能

SLSには4つの安全機能が組み込まれています。誤送信を防ぐための多層防御です。

#### 安全機能1: テストモード（DM_TEST_MODE）

| 項目 | 内容 |
|---|---|
| デフォルト | **ON**（テストモード有効） |
| 動作 | ホワイトリスト3名以外へのDMをすべてブロック |
| ホワイトリスト | `pojipojipoji`, `kantou1234`, `Nekomeem34` |
| ブロック時の状態 | `blocked_test_mode` |

テストモードがONの間は、ホワイトリスト以外のユーザーへのDMは `blocked_test_mode` ステータスで記録され、実際には送信されません。

#### 安全機能2: キャスト間分離ガード

hanshakun のChromeプロファイルから Risa_06 宛のDMを送信しようとすると、自動でブロックされます。

| 検証内容 | 説明 |
|---|---|
| セッションのキャストID | Chrome拡張が取得したStripchatユーザーID |
| 送信先のキャストID | `registered_casts` に登録されたStripchatユーザーID |
| 一致しない場合 | `blocked_identity_mismatch` でブロック |

> **なぜ必要か**: Stripchat APIはログイン中のアカウントからDMを送信します。
> hanshakun でログインしているのに Risa_06 の名前で送ろうとすると、
> 実際には hanshakun からDMが飛んでしまいます。これを防ぐ仕組みです。

#### 安全機能3: キャンペーンID必須ゲート

すべてのDM送信に `campaign_id` が必須です。
未設定の場合、`blocked_no_campaign` ステータスでブロックされます。

目的: すべてのDMに追跡タグを付けて、後から効果測定できるようにするため。

#### 安全機能4: 日次送信上限

| 項目 | 内容 |
|---|---|
| 上限 | **5,000件/日**（JST日付境界） |
| 対象 | `queued` + `sending` + `success` の合計 |
| 上限到達時 | HTTP 429エラー、新規送信を拒否 |

> **注意**: 上限はキャスト単位ではなく、アカウント全体の合計です。

#### その他の制限

| 制限 | 内容 |
|---|---|
| ユーザークールダウン | 同一キャスト→同一ユーザーへの再送信は**24時間ブロック** |
| 送信間隔 | 1通あたり**3秒**の待機時間 |
| キャンペーン形式 | `seg_*`, `bulk*`, `trigger_*`, `*_sched_*` 等の許可パターンのみ |

### 1.9 本番DM送信の手順

テストモードを解除して本番送信するための手順です。

#### 事前準備（送信前）

1. **テスト送信で確認**: テストモードONのまま、ホワイトリストユーザー（pojipojipoji等）で送信テスト
2. **dm_send_log で結果確認**: ステータスが `success` になっていることを確認
3. **Stripchat側で受信確認**: ホワイトリストユーザーのDM受信箱を開いて、メッセージ内容を確認

#### 本番送信

1. **Chrome拡張でセッション同期を確認**
   - 該当キャストのChromeプロファイルでStripchatにログインしていることを確認
   - Chrome拡張のポップアップで「セッション有効」が表示されていることを確認

2. **環境変数を変更して DM_TEST_MODE を解除**
   - Collector の `.env` ファイル:
     ```
     DM_TEST_MODE=false
     ```
   - Collector（dm-service）を再起動:
     ```bash
     cd C:\dev\livespot\collector
     pm2 restart dm-service
     ```

3. **SLS画面でDMを送信**（1.7の手順通り）

4. **送信状況をリアルタイム監視**
   - DMタブの送信ログで `queued` → `sending` → `success` の遷移を確認
   - エラーがあれば `error` ステータスとエラーメッセージを確認

5. **送信完了後、テストモードを復帰**
   - `.env` ファイルを元に戻す:
     ```
     DM_TEST_MODE=true
     ```
   - Collector を再起動:
     ```bash
     pm2 restart dm-service
     ```

> **重要**: 送信が完了したら、必ずテストモードをONに戻してください。
> テストモードOFFのまま放置すると、他のオペレーターが誤って本番送信してしまう危険があります。

### 1.10 送信結果の確認

#### dm_send_log のステータス一覧

| ステータス | 意味 | 対応 |
|---|---|---|
| `queued` | キュー待機中（未送信） | DMサービスが順次処理する。待つだけ |
| `sending` | 送信処理中 | 数秒以内に success/error に変わる |
| `success` | 送信完了 | 正常終了 |
| `error` | 送信失敗 | エラーメッセージを確認して対処 |
| `blocked_test_mode` | テストモードでブロック | テストモード解除が必要（本番送信時） |
| `blocked_no_campaign` | キャンペーンID未設定 | 再登録時にキャンペーンを設定 |
| `blocked_identity_mismatch` | キャスト不一致でブロック | 正しいChromeプロファイルでセッション同期 |

### 1.11 よくあるエラーと対処法

#### 「No active Stripchat session」

| 原因 | Stripchatのログインセッションが無効 |
|---|---|
| 対処 | 該当キャストのChromeプロファイルでStripchatにログイン → Chrome拡張でセッション同期 |

#### 「campaign_idが指定されていません」

| 原因 | DM送信時にキャンペーン名が空 |
|---|---|
| 対処 | メッセージ設定画面でキャンペーン名を入力してから送信 |

#### 「CAST_IDENTITY_MISMATCH」

| 原因 | Chromeでログイン中のキャストと、DM送信先のキャストが異なる |
|---|---|
| 対処 | 送信先キャストのChromeプロファイルに切り替えてStripchatにログイン |

#### 「1日あたりの送信上限(5,000件)に到達しました」

| 原因 | 本日の送信数が5,000件を超えた |
|---|---|
| 対処 | 翌日（JST 0:00リセット）まで待つ |

#### 送信が `queued` のまま動かない

| 原因 | DMサービスが停止している可能性 |
|---|---|
| 対処 | `pm2 status` でdm-serviceが動いているか確認。停止していれば `pm2 restart dm-service` |

## 2. 競合分析（SPY）

SPY機能は、Stripchatの公開チャットをリアルタイムで監視し、自社キャストの運営と他社キャストの分析に活用する機能です。

### 2.1 SPY一覧画面の見方

URL: `/spy`

画面上部に2つのメインタブがあります。

| タブ | 内容 |
|---|---|
| 🏠 **自社キャスト** | `registered_casts` に登録された自社キャストのデータ |
| 🔍 **他社キャスト** | `spy_casts` に登録された競合キャストのデータ |

各タブの中にさらにサブタブがあります。

#### 自社キャストのサブタブ

| サブタブ | 内容 |
|---|---|
| 📡 リアルタイム | チャットメッセージのリアルタイム表示 |
| 📋 キャスト一覧 | 自社キャストの一覧テーブル |
| 🤖 FBレポート | AI分析レポート一覧 |

#### 他社キャストのサブタブ

| サブタブ | 内容 |
|---|---|
| 📡 リアルタイム | チャットメッセージのリアルタイム表示 |
| 📋 キャスト一覧 | 他社キャストの一覧テーブル（タグ付き） |
| 📊 マーケット分析 | 複数キャストの横断分析（準備中） |
| 📦 型カタログ | ビジネスモデル別の分類（準備中） |

#### リアルタイム画面の構成

画面は3つのエリアに分かれています。

**上部: ステータスバー**

| 表示項目 | 説明 |
|---|---|
| 接続状態 | 🟢 監視中 / 🟡 一時停止 / 🔴 停止 |
| 最終受信 | 最後のメッセージからの経過時間 |
| MSG件数 | 画面内のメッセージ総数 |
| TIP合計 | チップ・ギフトのコイン合計（tk + 円換算） |
| USERS | ユニークユーザー数 |

自社タブでは追加で:
- **視聴者数**: 全体 / コイン保持 / その他の内訳（30秒ごと更新）

**左パネル: キャスト一覧**

各キャストが一覧表示され、以下の操作ができます:
- クリック: そのキャストのメッセージだけにフィルタ
- 目玉アイコン: そのキャストの表示/非表示を切り替え
- 🗑 ボタン: 今日のログを削除
- 🔍 ボタン（他社のみ）: キャスト詳細ページへ移動

**中央: メッセージ表示エリア**

チャットメッセージがリアルタイムで流れます。各メッセージには以下が表示されます:
- 時刻（HH:MM）
- ユーザー名（チップ/ギフト時は黄色で強調）
- トークン数（課金メッセージのみ）
- メッセージ本文
- VIPバッジ（該当時）

**右パネル: リアルタイム統計（自社のみ）**

| 項目 | 説明 |
|---|---|
| トップチッパー | 配信中のチップ上位5名（金銀銅バッジ付き） |
| アクティブユーザー | 直近5分以内にメッセージを送った人数 |
| チャット速度 | 1分あたりのメッセージ数 |
| 盛り上がり | 通常の1.5倍以上のチャット速度で 🔥 表示 |

#### キャスト監視状態（自社のみ）

各キャスト名の横に表示されるバッジで、配信状況を示します。

| バッジ | 条件 | 意味 |
|---|---|---|
| 🟢 LIVE | 最終メッセージから5分以内 | 配信中 |
| 🟡 IDLE | 5〜30分 | 配信しているが静か |
| 🔴 OFFLINE | 30分以上 | 配信していない |

### 2.2 メッセージフィルタの使い方

メッセージ表示エリアの上部にフィルタボタンがあります。

| ボタン | 対象メッセージ | 使いどころ |
|---|---|---|
| 💬 チャット | 一般的なチャットメッセージ | 会話の流れを見たいとき |
| 🪙 チップ | チップ、ギフト、グループ参加/終了 | 課金状況だけ追いたいとき |
| 🎤 音声 | 文字起こし（STT）メッセージ | キャストの発言を確認したいとき |
| 🚪 入退室 | 入室・退室メッセージ | ユーザーの出入りを追いたいとき |
| ⚙ システム | ゴール達成、視聴者数更新等 | 配信イベントを見たいとき |

- フィルタは複数選択可能（AND条件）
- フィルタ設定はブラウザに自動保存され、次回アクセス時に復元されます

**テキスト検索**: フィルタボタンの横にある検索ボックスにテキストを入力すると、ユーザー名またはメッセージ本文に一致するメッセージだけが表示されます（大文字小文字の区別なし）。

### 2.4 SPYキャスト詳細の見方

キャスト一覧で他社キャスト名をクリック、または🔍ボタンを押すと詳細ページに遷移します。

URL: `/spy/[キャスト名]`（例: `/spy/fantasy_JP`）

4つのタブで構成されています。

#### タブ1: 概要（📊）

キャストの基本情報と集計値を一覧表示します。

**統計カード（4列）**

| カード | 内容 | データソース |
|---|---|---|
| メッセージ | チャットメッセージ総数 | chat_logs の行数 |
| チップ回数 | チップ + ギフトの出現回数 | message_type が tip/gift の行数 |
| コイン合計 | 累計コイン数 + 日本円換算 | tokens カラムの合計 |
| ユニークユーザー | 重複なしのユーザー数 | DISTINCT(username) |

**キャスト情報**

| 項目 | 説明 |
|---|---|
| カテゴリ | 人妻 / 女子大生 / ギャル 等（手動設定） |
| フォーマットタグ | 配信スタイルの簡潔な説明（手動設定） |
| メモ | 運営メモ（自由記入） |
| 最終活動 | 最後にチャットメッセージがあった時刻 |

**型情報（設定済みの場合）**

| 項目 | 説明 |
|---|---|
| 型名 | ビジネスモデルの分類名 |
| ルート | 「収穫型」（短期大量課金）or「育成型」（長期関係構築） |
| 収益パターン | チケチャ回転型 / パブ重視型 / ハイブリッド |
| 売上レンジ | 1セッション当たりの期待売上幅 |

**トップチッパー**: 課金額上位のユーザーをランキング表示（金銀銅バッジ付き）。

**最近のアクティビティ**: 直近15件のチャットメッセージをログ形式で表示。

#### タブ2: 配信ログ（📺）

過去の配信セッション一覧をテーブルで表示します。

| カラム | 内容 |
|---|---|
| 日付 | 配信日（YYYY/M/D） |
| 時間 | 開始〜終了時刻 + 配信時間（分） |
| タイトル | 配信タイトル（設定されている場合） |
| MSG | セッション内のメッセージ数 |
| TIP | チップ/ギフトの回数 |
| COINS | 合計コイン数（チップ + チケットショー推定） |
| USERS | セッション内のユニークユーザー数 |

**チケットショーの表示**: チケットショーが検出された場合、COINS列に内訳が表示されます。
- 例: `チップ 2.5K + チケット 1.8K (2回)`

> **注意**: COINSの金額はSPYチャットで観測できたチップ/ギフトのみです。
> プライベート配信やCam2Camの収入は含まれません（後述の「分かること/分からないこと」参照）。

#### タブ3: ユーザー分析（👥）

キャストの応援ユーザーをランキング形式で表示します。

| カラム | 内容 |
|---|---|
| # | 順位（🥇🥈🥉） |
| ユーザー | ユーザー名（クリックでユーザー詳細へ） |
| ステータス | 🟢 active / 🔵 new / 🟠 at_risk / 🔴 churned / 灰 free |
| COINS | 累計コイン数 |
| TIP回数 | チップ/ギフトの出現回数 |
| 最終TIP | 最後にチップした時期（相対表示） |
| 初回TIP | 初めてチップした日 |

**ステータスの意味**

| ステータス | 意味 |
|---|---|
| 🟢 active | 直近も継続的に課金している |
| 🔵 new | 最近初めて課金した新規ユーザー |
| 🟠 at_risk | 以前は課金していたが最近減少 |
| 🔴 churned | 長期間課金がない離脱ユーザー |
| 灰 free | チャットはするが課金ゼロ |

#### タブ4: フォーマット分析（📋）

現在は準備中のプレースホルダーです。以下のメタデータのみ表示されます:
- カテゴリ
- フォーマットタグ
- メモ

### 2.5 競合データの読み方

SPYで収集できるデータには明確な限界があります。分析時に「分かること」と「分からないこと」を理解しておくことが重要です。

#### 分かること

| 項目 | 精度 | 説明 |
|---|---|---|
| 配信中かどうか | 高 | チャットメッセージの有無でほぼリアルタイムに判定 |
| 配信時間帯パターン | 高 | 過去のセッション開始/終了時刻から曜日・時間帯の傾向が分かる |
| チャットの活気度 | 高 | メッセージ数、チャット速度で客観的に測定可能 |
| 公開チップ/ギフトの金額 | 高 | チャット内で表示されるチップとギフトの金額を正確に記録 |
| チケットショーの有無 | 中 | 連続した高額チップパターンから推定（アルゴリズムベース） |
| トップユーザーの動向 | 中 | チップ額上位のユーザー名と金額。ただし匿名チップは不可 |
| ユーザーのリテンション | 中 | 最終チップ日の差分でactive/churned等を分類 |

#### 分からないこと

| 項目 | 理由 |
|---|---|
| **プライベート配信の収入** | プライベートチャットはSPY不可。公開チャットのみ監視対象 |
| **Cam2Camの収入** | 同上 |
| **グループチャットの収入** | 一部検出可能だが不完全 |
| **チケット価格** | チケットショー開始のメッセージからショーの存在は分かるが、チケット1枚の価格は不明 |
| **視聴者数** | 他社キャストの視聴者数APIは認証が必要なため取得不可 |
| **ユーザーのレベル/リーグ** | チャットメッセージには含まれない（別途API必要） |
| **ファン登録数** | API非公開 |
| **キャストの手取り金額** | スタジオとの分配率が不明 |
| **配信の映像内容** | テキストチャットのみ。映像・音声は対象外 |

#### SPYデータの実際の活用方法

1. **配信時間帯の競合回避**: 競合キャストの配信パターンを把握し、空いている時間帯を狙う
2. **人気の目安**: メッセージ数やチップ額で競合の人気度を比較（ただし公開分のみ）
3. **ユーザーの共有**: 他社キャストのトップユーザーが自社キャストにも来ているか確認
4. **型（ビジネスモデル）の研究**: 成功している競合キャストの配信スタイルを参考にする
5. **チケットショー戦略**: 競合のチケットショー頻度・売上を参考に自社の戦略を立てる

> **重要**: SPYの数値は「公開チャットで見えた分だけ」です。
> 実際の売上は、SPYで見える金額の **数倍〜10倍** になることがあります。
> 正確な自社売上は Coin API（キャスト詳細ページの「売上」タブ）で確認してください。
