# SLS 操作マニュアル Part 1: 初期セットアップ + キャスト管理

> 対象: SLS（Strip Live Spot）を初めて使うオペレーター
> 最終更新: 2026-03-06

---

## 1. 初期セットアップ

SLSを使い始めるには、以下の4つを順番に準備します。

1. キャストごとのChromeプロファイル
2. Chrome拡張のインストール
3. Stripchatへのログイン
4. SLSへのログイン

### 1.1 Chromeプロファイルの準備

SLSではキャストごとに**別々のChromeプロファイル**を使います。
これはStripchatのログインセッション（Cookie）がキャストごとに異なるためです。
1つのプロファイルで複数キャストにログインすると、Cookieが混在してDM送信先を間違える危険があります。

#### 手順（例: Risa_06用プロファイルを作成）

1. Chromeを開き、右上のプロフィールアイコン（丸いアバター）をクリック
2. 「+ 追加」をクリック
3. プロフィール名を入力: `Risa_06`
   - アイコンや色は好みで設定してOK
   - Googleアカウントとの紐付けは不要（「ログインしない」を選択）
4. 「完了」をクリック

同様に `hanshakun` 用のプロファイルも作成します。

#### 確認方法
- Chromeのタイトルバーまたはプロフィールアイコンに「Risa_06」と表示されていればOK

#### 失敗した場合
- プロファイル名は後から変更可能: `chrome://settings` → 「Chromeプロフィールをカスタマイズ」
- プロファイルが見つからない場合: `chrome://settings/manageProfile` から一覧を確認

> **ルール**: Stripchatの操作は必ず該当キャストのプロファイルで行うこと。
> プロファイルを間違えると、別のキャストのCookieが上書きされます。

---

### 1.2 Chrome拡張のインストール

SLS Chrome拡張（Strip Live Spot - Auth Exporter）は、StripchatのログインセッションをSLSに連携するための拡張機能です。
**各キャストのChromeプロファイルそれぞれに**インストールします。

#### 手順

1. 該当キャストのChromeプロファイルでChromeを開く
2. アドレスバーに `chrome://extensions` と入力してEnter
3. 右上の「デベロッパーモード」をONにする（トグルスイッチ）
4. 「パッケージ化されていない拡張機能を読み込む」をクリック
5. フォルダ選択ダイアログで `C:\dev\livespot\chrome-extension` を選択
6. 「フォルダーの選択」をクリック

#### 確認方法
- 拡張機能一覧に「Strip Live Spot - Auth Exporter」（v3.0.0）が表示される
- Chromeのツールバー（右上）にダイヤモンド型のアイコンが表示される
  - まだSLSにログインしていない場合、アイコンに特にバッジは表示されません

#### 失敗した場合
- 「マニフェストファイルが見つかりません」→ 選択したフォルダが間違っている。`chrome-extension` フォルダ直下に `manifest.json` があるか確認
- エラーが表示される場合 → `chrome://extensions` のエラーボタンを押してエラー内容を確認

> **注意**: `hanshakun` 用のプロファイルにも同じ手順でインストールしてください。
> 2つのプロファイルにそれぞれ1回ずつ、合計2回インストールが必要です。

---

### 1.3 Stripchatへのログイン

Chrome拡張がStripchatのセッションCookieを取得するには、まずStripchatにログインしている必要があります。

#### 手順（例: Risa_06用プロファイル）

1. **Risa_06用Chromeプロファイル**でブラウザを開く（プロファイルを間違えないこと！）
2. `https://stripchat.com` にアクセス
3. 右上の「ログイン」からRisa_06のアカウントでログイン
   - メールアドレスとパスワード、またはStripchat IDで認証
4. ログイン完了後、Stripchatのトップページが表示されることを確認

#### 確認方法
- Stripchatの右上にユーザー名（Risa_06）が表示されている
- `https://stripchat.com/user/dashboard` にアクセスしてダッシュボードが表示される

#### 失敗した場合
- 「アカウントがロックされている」→ Stripchatサポートに連絡
- 2段階認証が求められる → 該当キャストの認証コードを入力

> **重要**: hanshakun用のプロファイルではhanshakuのアカウントでログインしてください。
> **絶対にプロファイルとキャストアカウントを間違えないこと。**

### 1.4 SLSへのログイン（Chrome拡張）

Stripchatにログインした状態で、Chrome拡張からSLSにログインします。
これにより、Chrome拡張がStripchatのCookieをSLSのサーバーに安全に送信できるようになります。

#### 手順

1. Chromeツールバーの **SLS拡張アイコン**（ダイヤモンド型）をクリック
   - ポップアップが開き、「未接続」と赤い点が表示される
2. ログインフォームに以下を入力:
   - **メールアドレス**: SLSに登録済みのメールアドレス（例: `admin@livespot.jp`）
   - **パスワード**: SLSのパスワード
3. 「ログイン」ボタンをクリック

#### 確認方法（重要！）
ログイン成功後、ポップアップの表示が以下のように変わります:

1. **接続状態**: 右上の点が **緑色** に変わり、「接続中」と表示される
2. **アカウント選択**: 「アカウント」ドロップダウンが表示される（自動選択済み）
3. **キャスト身元表示**: 緑色の枠で以下のいずれかが表示される:
   - `ログイン中: Risa_06` → 正常。このプロファイルでRisa_06のCookieが検出されている
   - `ログイン中: hanshakun` → 正常（hanshakun用プロファイルの場合）

#### 異常な表示と対処法

| 表示 | 意味 | 対処法 |
|------|------|--------|
| 緑点 +「ログイン中: Risa_06」 | 正常 | そのまま使用OK |
| 黄色枠 +「Stripchat未ログイン」 | StripchatにログインしていないCookieがない | 1.3の手順でStripchatにログインし直す |
| 黄色枠 +「不明なアカウント (ID: 12345)」 | Stripchatにログインしているが、SLSにキャスト登録されていない | 2章の手順でキャストを登録する |
| 赤枠 +「複数アカウント検出!」 | Cookieが混在している（危険） | 「リセット」ボタンを押してCookieを削除し、正しいキャストで再ログイン |
| 赤点 +「未接続」 | SLSにログインできていない | メールアドレスとパスワードを確認して再ログイン |

> **「リセット」ボタン**: Stripchatの認証Cookieを削除します。
> 押した後はStripchatに再ログインが必要です。
> Cookieが混在した場合（複数アカウント検出時）に使います。

### 1.5 SLSウェブ画面へのログイン

Chrome拡張とは別に、SLSのウェブ管理画面にもログインします。
キャスト管理、売上分析、DM管理などはすべてこのウェブ画面で行います。

#### 手順

1. ブラウザで `https://livespot-rouge.vercel.app/login` にアクセス
   - どのChromeプロファイルからでもOK（ウェブ画面はキャスト別にする必要なし）
2. メールアドレスとパスワードを入力
3. 「ログイン」ボタンをクリック

#### 確認方法
- ログイン成功後、ダッシュボード（コントロールセンター）が表示される
- 左側にサイドバーメニューが表示される（キャスト、SPY、分析 など）

#### 失敗した場合
- 「メールアドレスまたはパスワードが正しくありません」→ 入力内容を確認。SLS登録時のメールアドレスとパスワードを使用
- 画面が白いまま → ブラウザのキャッシュをクリアして再試行

### 1.6 初回データ同期

SLSの売上データやユーザー情報は、Stripchatから自動的に取得されます。
初回はデータが空の状態なので、以下の方法でデータ同期を開始します。

#### コイン履歴の同期（自動）

コイン履歴の取得はCollector（サーバーサイドのバッチ処理）が自動で行います。
Chrome拡張がStripchatにログインした状態でSLSにも接続していれば、以下が自動で実行されます:

1. **セッションCookieのエクスポート**: Chrome拡張が定期的にStripchatのセッションCookieをSLSサーバーに送信（5分間隔）
2. **Collectorによる取得**: サーバー側のCollectorがそのCookieを使ってStripchat Earnings APIからコイン履歴を取得
3. **データベースに保存**: coin_transactionsテーブルにUPSERT

#### 同期状態の確認

SLSウェブ画面のキャスト詳細ページ（`/casts/キャスト名`）にアクセスすると、概要タブにコイン同期の最終実行時刻が表示されます。

#### SPYチャット監視（自動）

Chrome拡張がStripchatのページを開いている間、チャットメッセージ（tip、gift、通常メッセージ）が自動的に収集されます。
加えて、Collector（サーバーサイド）もWebSocket経由でチャットを収集しているため、Chrome拡張を閉じていてもデータは蓄積されます。

#### 初回同期が完了するまで

- コイン履歴: Collectorの次回実行時に取得（最大6時間待ち、通常は配信終了後に自動実行）
- SPYチャット: Stripchatページを開いた瞬間からリアルタイム収集開始
- ユーザー名簿: コイン履歴取得時に自動更新

> **注意**: 初回はデータが空なので、ダッシュボードやグラフに何も表示されません。
> コイン同期が1回実行されるとデータが表示され始めます。

## 2. キャスト管理

### 2.1 キャスト管理画面へのアクセス

SLSウェブ画面にログインした状態で:

1. 左サイドバーの「管理」セクションにある「キャスト」をクリック
   - または直接 `https://livespot-rouge.vercel.app/admin/casts` にアクセス
2. キャスト管理画面が表示される

この画面には2つのタブがあります:
- **自社キャスト**: 自分のエージェンシーに所属するキャスト（registered_casts）
- **他社キャスト**: 競合分析用に監視するキャスト（spy_casts）

---

### 2.2 自社キャストの登録

#### 手順

1. キャスト管理画面の右上にある「+ 自社キャスト登録」ボタンをクリック
   - `/admin/casts/new` ページに遷移する
2. フォームに情報を入力:

| 項目 | 必須 | 説明 | 例 |
|------|------|------|------|
| **キャスト名** | **必須** | StripchatのユーザーID（URLと一致する名前） | `Risa_06` |
| 表示名 | 任意 | UI上に表示する日本語名 | `りさ` |
| プラットフォーム | 任意 | 配信プラットフォーム（デフォルト: Stripchat） | `Stripchat` |
| StripchatモデルID | 任意 | API連携に使用する数値ID（後から設定可） | `12345678` |
| ジャンル | 任意 | プリセットから選択 | `女性単体` |
| ランク | 任意 | プリセットから選択 | `中堅` |
| カテゴリ | 任意 | プリセットから選択 | `お姉さん` |
| スクリーンショット間隔 | 任意 | 配信中の自動スクショ間隔（デフォルト: 5分） | `5分` |
| GCレート | 任意 | グループチャット1分あたりのトークン消費（デフォルト: 12） | `12` |
| レベニューシェア分配率 | 任意 | ネット売上に対するキャストへの分配率（デフォルト: 50%） | `50` |
| メモ | 任意 | 自由メモ | `火・木・土 21時配信` |

3. 「キャストを登録」ボタンをクリック
4. 成功するとキャスト管理一覧に戻る

#### 確認方法
- キャスト管理画面の「自社キャスト」タブに登録したキャストが表示される
- 「有効」カウンターが1増えている

#### 失敗した場合
- 「Risa_06 は既に登録済みです」→ 同じキャスト名が既に登録されている。一覧を確認
- 一度無効化したキャストを同じ名前で登録しようとすると、自動的に再有効化される

> **重要**: キャスト名はStripchatのプロフィールURL（`https://stripchat.com/Risa_06`）と一致する名前を正確に入力してください。
> 大文字・小文字・アンダースコアもすべて一致している必要があります。

---

### 2.3 キャスト情報の編集

#### 手順

1. キャスト管理画面で、編集したいキャストの行の右端にある「編集」ボタンをクリック
2. テーブル行がインライン編集フォームに展開される
3. 変更したい項目を編集:
   - 表示名、プラットフォーム、StripchatモデルID、StripchatユーザーID
   - 分配率、スクリーンショット間隔、GCレート
   - ジャンル、ランク、カテゴリ
   - メモ
4. 「保存」ボタンをクリック

#### StripchatモデルIDの調べ方

StripchatモデルIDは、StripchatのAPI連携（視聴者リスト取得など）に必要な数値IDです。

1. Stripchatでキャストのプロフィールページを開く: `https://stripchat.com/Risa_06`
2. ブラウザの開発者ツール（F12）を開く
3. Consoleタブに以下を入力:
   ```javascript
   fetch('/api/front/v2/models/username/Risa_06').then(r=>r.json()).then(d=>console.log('modelId:', d.user.id))
   ```
4. 表示された数値がモデルID

#### StripchatユーザーIDの調べ方

ユーザーIDはCookieから自動取得されますが、手動で確認する場合:

1. Stripchatにログインした状態で開発者ツールを開く（F12）
2. Application → Cookies → `https://stripchat.com`
3. `stripchat_com_userId` の値がユーザーID
4. 見つからない場合は、AMP Cookieをデコード:
   - `AMP_` で始まるCookieの値をBase64デコードし、JSONの`userId`フィールドを確認

---

### 2.4 キャストの無効化・削除

#### 無効化（一時停止）

1. キャスト管理画面で、対象キャストの行の「無効化」ボタンをクリック
2. 確認ダイアログで「OK」をクリック
3. キャストは「無効」リストに移動し、データ収集が停止する

**無効化の効果**:
- Collectorがこのキャストのデータを収集しなくなる
- 既存のデータは保持される
- 「有効化」ボタンで復活可能

#### 完全削除

1. 無効化されたキャストの行に「削除」ボタンが表示される
2. 「削除」ボタンをクリック
3. 確認ダイアログ（「この操作は取り消せません」）で「OK」をクリック

**削除の効果**:
- registered_castsからレコードが完全に削除される
- コスト設定（cast_cost_settings）も同時に削除される
- **注意**: coin_transactions, spy_messages等の関連データは削除されない（孤児データとして残る）

---

### 2.5 他社キャストの登録（SPY監視用）

競合分析のために他社キャストを監視対象に追加できます。

#### 手順

1. キャスト管理画面で「他社キャスト」タブをクリック
2. 画面上部の「キャスト名」入力欄に、StripchatのユーザーID（URL名）を入力
   - 例: `fantasy_JP`
3. 「追加」ボタンをクリック

#### 他社キャストで設定できる項目

編集ボタンをクリックすると、以下が設定可能:
- 表示名、StripchatモデルID
- ジャンル、ランク、カテゴリ、フォーマットタグ
- スクリーンショット間隔
- 自動監視（auto_monitor）のON/OFF
- メモ

#### フィルタ機能
- **ジャンル**: ドロップダウンで特定ジャンルのキャストのみ表示
- **活動停止を非表示**: 長期間配信のないキャストを非表示

## 付録: registered_castsテーブルの全カラム

キャスト登録時にDBに保存されるフィールドの一覧です。

| カラム | 型 | 説明 |
|--------|------|------|
| id | UUID (PK) | 自動生成 |
| account_id | UUID (FK) | 所属アカウント |
| cast_name | TEXT | Stripchatユーザー名（**必須**） |
| display_name | TEXT | 表示用日本語名 |
| platform | TEXT | 配信プラットフォーム（デフォルト: stripchat） |
| stripchat_model_id | TEXT | Stripchat APIのモデルID |
| stripchat_user_id | BIGINT | StripchatのユーザーID（Cookie連携で自動取得） |
| stripchat_url | TEXT | プロフィールURL（自動生成: `https://stripchat.com/{cast_name}`） |
| genre | TEXT | ジャンル |
| benchmark | TEXT | ランク |
| category | TEXT | カテゴリ |
| screenshot_interval | INTEGER | スクリーンショット間隔（分、デフォルト: 5） |
| gc_rate_per_minute | NUMERIC | GCレート（tk/分、デフォルト: 12） |
| notes | TEXT | 自由メモ |
| is_active | BOOLEAN | 有効/無効（デフォルト: true） |
| created_at | TIMESTAMPTZ | 作成日時 |
| updated_at | TIMESTAMPTZ | 更新日時 |

関連テーブル:
- **cast_cost_settings**: キャストごとの分配率設定（revenue_share_rate, platform_fee_rate, token_to_usd）
- **spy_casts**: 他社キャスト管理テーブル（構造はregistered_castsと類似、format_tag, auto_monitor, is_extinct が追加）
