# 品質チェック手順

## トリガー
コード変更後、コミット前に実行

## チェック項目

### コード品質
- [ ] ビルドが通るか（tsc --noEmit / npm run build）
- [ ] テストが通るか（既存テストがある場合）
- [ ] lint/formatエラーがないか

### セキュリティ
- [ ] .envファイルをコミットしていないか
- [ ] APIキー・トークンがハードコードされていないか
- [ ] SQLインジェクション・XSSの脆弱性がないか

### 設計原則チェック
- [ ] 全分析画面にアクションボタンがあるか（SLSの場合）
- [ ] データ原則: 2/15以前のデータを使っていないか
- [ ] coin_transactions.tokensで集計しているか（amountは不可）
- [ ] 「経験豊富な人が見てもOKと言えるか？」

### 理念チェック
- [ ] この変更は理念に沿っているか
- [ ] 技術要件だけをゴールにしていないか
